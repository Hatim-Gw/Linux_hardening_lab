# SSH
It's an abbreviation of Secure Shell, and it's been used to connect client hosts to Ubuntu servers to administer the servers remotely.
The privilege of using it instead of the old **telnet** is because of the encryption process between the server and the client.

# Steps:
1. checking SSH status using the command

**sudo systemctl status ssh**

3. Backup the configuration file and change the default values of port&permissions&athentications


**sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak**  (this command for the backup)

Then, edit the values in sshd_config
**sudo nano /etc/ssh/sshd_config**
