
# description
Making sure SSH root login is disabled, checking the members of the sudo group, reviewing the sudo policy, then installing *libpam-pwquality* to add rules for users' passwords, eventually checking if the tool is working by adding a new user.

# Commands
- sudo grep -i "permitrootlogin" /etc/ssh/sshd_config
- getent group sudo
- sudo visudo
- sudo apt install libpam-pwquality -y
- sudo nano /etc/security/pwquality.conf
- sudo nano /etc/pam.d/common-password
- sudo adduser


# steps performed:
1. I checked the root login over ssh if it's disabled or enabled using the command:
    - sudo grep -i "permitrootlogin" /etc/ssh/sshd_config
   it showed **No**, so remote users can't log in as the root

2. reveiwed the users of the group **sudo** to know who has the privileges to control the system.
   - getent group sudo
  
3. checked the default values of */etc/sudoers* using:
   - sudo visudo

4. installed libpam-pwquality tool to add more complexity to password rules.
  - sudo apt install libpam-pwquality -y

5. modified the configuration file of the tool */etc/security/pwquality.conf* using:
   - sudo nano /etc/security/pwquality.conf

   I added a rule to have at least one lowercase letter(lcredit = -1), one uppercase letter(ucredit = -1), and one digit letter(dcredit = -1).
